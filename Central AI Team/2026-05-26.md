# 📅 დღიური ჩანაწერი: 26 მაისი, 2026

დღევანდელი დღე იყო უმნიშვნელოვანესი პროექტისთვის **Repair Instruction Reader (სარემონტო ინსტრუქციების მკითხველი)**. ჩვენ წარმატებით დავასრულეთ ფრონტენდისა და ბექენდის მასშტაბური განახლებები, მოვაგვარეთ კვოტის პრობლემები და სრულად განვაახლეთ Obsidian-ის პირადი ბაზა (Vault).

## 🚀 დღევანდელი მიღწევების შეჯამება (Key Accomplishments)

### 1. ⚙️ საიდუმლო საინჟინრო რეჟიმი (Admin Mode & Settings Screen)
* **საიდუმლო გააქტიურება:** პარამეტრების კბილანას ხატულა (⚙️) დამალულია ტექნიკოსებისგან. მისი გამოჩენა შესაძლებელია **პორშეს ლოგოზე 3-ჯერ სწრაფი დაწკაპუნებით (Triple Click)** ან ბმულში `?admin=true` პარამეტრის გადაცემით.
* **უსაფრთხოების ფარი:** კბილანას გამოსაჩენად საჭიროა საინჟინრო პაროლის შეყვანა: `Suffering1@`.
* **Plug & Play მუშაობა:** ნაგულისხმევი API მისამართი შევცვალეთ live Render სერვერით, რაც ნიშნავს, რომ ნებისმიერი ახალი მომხმარებელი ავტომატურად უკავშირდება ცოცხალ სერვერს ყოველგვარი ხელით კონფიგურაციის გარეშე.

### 2. ⚡ კვოტის (429 Rate Limit) დაძლევა და SHA-256 ქეშირება
* **SHA-256 ფაილების ქეშირება:** ბექენდში დაინერგა ატვირთული PDF ფაილების ჰეშირება. თუ ტექნიკოსი ხელახლა ატვირთავს იდენტურ ფაილს, პასუხი ბრუნდება **50 მილიწამში** ლოკალური ქეშიდან Gemini-ს გამოძახების გარეშე.
* **API გასაღებების აუზი და როტაცია (Key Pooling & Rotation):** Google AI Studio-დან დავამატეთ **3 დამოუკიდებელი უფასო Gemini API გასაღები** როტაციის აუზში. ბექენდი ავტომატურად ატრიალებს გასაღებებს ყოველი მოთხოვნისას და 429 (Too Many Requests) შეცდომის შემთხვევაში ფონურად გადადის მომდევნო გასაღებზე.
* **ინტერფეისის თარგმანი:** 429 შეცდომისას ფრონტენდზე გამოდის მეგობრული ქართული შეტყობინება, რომელიც მომხმარებელს სთავაზობს პირადი გასაღების შეყვანას.

### 3. ☁️ Supabase Storage ინტეგრაცია და დუბლირების პრევენცია
* **Kong API Gateway-ს ავტორიზაცია:** მოგვარდა კრიტიკული ხარვეზი — REST API-ს მეშვეობით ატვირთვისას დავამატეთ `"apikey"` ჰედერი, რამაც წარმატებით გაიარა Supabase-ის Kong Gateway.
* **დუბლირების თავიდან აცილება:** ფაილების ატვირთვამდე ბექენდი აკეთებს `HEAD` მოთხოვნას Supabase-ში. თუ იდენტური მოდელისა და შეკეთების manual უკვე არსებობს, სისტემა ტოვებს ატვირთვას, რაც იცავს ბაზას გადავსებისგან.
* **ფაილების სუფთა დასახელებები:** PDF ფაილებს ავტომატურად ერქმევა მოდელისა და შეკეთების ინგლისური დასახელებები (სპეციალური სიმბოლოებისგან გასუფთავებული), მაგ: `Cayenne_Replace_Silencer.pdf`.

### 4. 🏎️ პრემიუმ ესთეტიკის განახლებები (Porsche Cockpit Polish)
* **Driving Mode Selector Dial:**navbar-ში დაემატა პორშეს მართვის რეჟიმების გადამრთველი: **N (Normal), S (Sport - Guards Red), და GT (Track mode - Acid Green)**. რეჟიმის შეცვლა მომენტალურად ცვლის საიტის ფერებსა და ატმოსფეროს.
* **Circular Sport Chrono Timer:** ფაილის დამუშავებისას, ჩვეულებრივი Loading ბარის ნაცვლად, ტრიალებს წრიული **Sport Chrono წამზომი (Chronometer)**, რომელიც მილიწამების სიზუსტით ითვლის დროს.
